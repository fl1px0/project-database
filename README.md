# Project-database
Мой проект  "Доставка еды",включающий в себя реляционную базу данных и ER-диаграмму
# Cтруктура базы данных
База данных включает следующие основные таблицы:
 • Customers — хранит информацию о клиентах сервиса
 • Restaurants — содержит данные о ресторанах, подключённых к сервису
 • MenuItems — хранит сведения о блюдах и их стоимости
 • Orders — содержит информацию о заказах клиентов
 • OrderItems — отражает состав каждого заказа
 • Couriers — хранит данные о курьерах
 • Deliveries — фиксирует процесс доставки заказов
# Связи между таблицами базы данных
 1. Customers — Orders
Тип связи: один ко многим 
 2. Restaurants — MenuItems
Тип связи: один ко многим 
 3. Restaurants — Orders
Тип связи: один ко многим 
 4. Orders — OrderItems
Тип связи: один ко многим 
 5. MenuItems — OrderItems
Тип связи: один ко многим 
 6. Couriers — Deliveries
Тип связи: один ко многим 
 7. Orders — Deliveries
Тип связи: один к одному

Связи между таблицами реализованы с использованием первичных и внешних ключей
# Заполнение данных
База данных заполнена тестовыми данными в следующем объёме:
 • Customers — 5 записей  (данные о клиентах сервиса доставки еды)
 • Restaurants — 4 записи  (рестораны различных кухонь)
 • MenuItems — 12 записей  (блюда и напитки из меню ресторанов)
 • Orders — 6 записей  (заказы клиентов в различных статусах)
 • OrderItems — 14 записей  (состав заказов, включающий несколько блюд)
 • Couriers — 4 записи  (курьеры с разными типами транспорта)
 • Deliveries — 5 записей  (доставки заказов, включая завершённые и активные)
